# Decision Log

## Purpose

Provide a dedicated place for architecture-specific decisions that emerge from the maps, references, and future diagram work.

## Core Rules

- Record durable architecture decisions here when they are specific to the architecture mapping layer.
- Keep entries aligned with canonical project decisions elsewhere in the repo.
- Avoid duplicating decision records without a clear reason.
- Link major decisions back to the maps and references they affect.

## Relationships

- Architecture decisions should connect to product hierarchy, workflow, security, and data diagrams.
- This log should support the broader project decision history without competing with it.
- Decision entries should inform future scaffolding and implementation readiness.

## Open Questions

- Which architecture decisions should live here versus in the main decision docs?
- What format should future architecture decision entries follow?
- Which diagrams should require a decision record when changed materially?

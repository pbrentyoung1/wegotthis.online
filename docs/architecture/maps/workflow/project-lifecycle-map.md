# Project Lifecycle Map

## Purpose

Describe the project lifecycle from intake through archive and clarify the major state transitions.

## Core Rules

- Keep request lifecycle separate from project lifecycle.
- Use Planned between Approved and Active when work is accepted but production has not started.
- Use In Review for project-level review visibility.
- Do not use Waiting for Approval as a project status.
- Do not move projects directly from published to archived.
- Require closeout before archive.
- Keep lifecycle definitions understandable to non-technical stakeholders.

## Relationships

- Projects may begin from accepted requests.
- Projects contain deliverables that may move through their own production states.
- Project lifecycle events should connect to communication, activity, and notification behavior.
- Closeout should connect project history, assets, approvals, and archive preparation.

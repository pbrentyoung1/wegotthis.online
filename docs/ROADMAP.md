# Roadmap

## Purpose

Provide a high-level planning path from repository setup through MVP and later product expansion.

## Current Decisions

- Phase 1: Establish documentation, decisions, and project structure.
- Phase 2: Finalize product scope, workflows, and Figma direction.
- Phase 3: Scaffold the Laravel, Vue, Inertia, PostgreSQL application.
- Phase 4: Build MVP modules around projects, requests, tasks, approvals, dashboard, calendar, and media library.
- Phase 5: Expand into vendors, print, signage, social content, templates, and advanced collaboration.

## Open Questions

- What exact modules belong in MVP?
- Which church workflow should be modeled first?
- What is the target date for a first internal demo?

## Notes

- Roadmap items should stay tied to documented product decisions.
- Avoid adding implementation commitments before architectural decisions are recorded.

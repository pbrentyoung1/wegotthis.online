# Calm Software Principles

## Purpose

Document the product principle that wegotthis.online should help and get out of the way.

## Current Decisions

- Working decision: the platform should reduce complexity rather than add overhead.
- The software should create calm, not complexity.

## Principles

- Create calm, not complexity.
- Guide instead of obstruct.
- Reduce decisions.
- Reduce interruptions.
- Make invisible work visible.
- Preserve institutional memory.
- Favor clarity over customization.
- Use opinionated defaults with flexible overrides.
- Keep workflows understandable.
- Reduce administrative overhead.
- Avoid building a system that requires more management than the work it replaces.
- Help and get out of the way.

## Product Implications

- Prefer clear defaults over excessive settings.
- Use templates to reduce setup burden.
- Keep stakeholder and vendor review flows simple.
- Make status visible without requiring meetings or messages.
- Keep mobile review and approval flows fast.
- Avoid turning reporting, closeout, or brand review into unnecessary bureaucracy.

## Open Questions

- Which customization requests should be rejected to preserve product clarity?
- Where do churches need flexibility most?
- Which workflows can be simplified through opinionated defaults?

## Notes

- This principle should influence product design, architecture, permissions, templates, and notifications.

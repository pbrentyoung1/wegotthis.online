# Strategic Context and Goal Alignment

## Purpose

Define how ForWorship Creative connects communications and production work to ministry goals, campaign strategy, recurring production rhythms, and measurable outcomes.

This document clarifies the difference between Initiatives, Campaigns, Programs / Content Streams, Projects, Deliverables, and Tasks.

The goal is to help communications teams align work with purpose without forcing every request into the wrong container.

## Core Principle

ForWorship Creative should distinguish strategic context from operational work.

Strategic context explains what the work supports.

Operational work is where production happens.

The core operational model remains:

```text
Project
  -> Deliverable
    -> Task
```

A Project may support one or more strategic contexts, such as an Initiative, Campaign, Program / Content Stream, Sermon Series, Season, Ministry Area, or Topic.

## Why This Matters

Church communications work often fails when the production team receives isolated deliverable requests without understanding the goal.

Goal alignment helps Communications answer:

- Why are we doing this?
- What larger goal does this support?
- Who is the audience?
- What response are we trying to create?
- Is this part of a campaign, recurring production rhythm, sermon series, season, or measurable initiative?
- Which deliverables are strategically necessary?
- Which requested ideas should be accepted, modified, deferred, or declined?

The system should help Communications connect ministry needs to creative strategy while keeping the user experience simple.

## Strategic Contexts

Strategic contexts are not all the same kind of object. They represent different ways work can be aligned, grouped, reported, or understood.

### Initiative

An Initiative is a broad strategic goal or measurable priority.

It answers:

> What larger measurable outcome are we trying to move over time?

Examples:

- Increase daily sermon clip views by 50% by the end of the year.
- Grow volunteer participation by 20% this ministry year.
- Improve first-time guest follow-up engagement.
- Increase online giving participation.
- Build awareness of family ministry.
- Improve consistency of weekly digital content.

Initiatives are usually:

- Long-running.
- Measurable.
- Cross-campaign.
- Cross-project.
- Leadership-level or department-level.
- Used for reporting and strategic planning.
- Not necessarily tied to one event, season, or launch.

Recommended Initiative fields:

| Field | Purpose |
|---|---|
| Name | Strategic objective name. |
| Goal | What the organization is trying to accomplish. |
| Success metric | How progress is measured. |
| Baseline | Starting point. |
| Target | Desired measurable outcome. |
| Start date | When the initiative begins. |
| Target/end date | When success should be evaluated. |
| Owner | Who is accountable. |
| Ministry/department | Who it serves or belongs to. |
| Audience | Who the initiative affects. |
| Strategy notes | Broad approach. |
| Related campaigns/projects | Work that supports the goal. |
| Status | Planned, Active, Paused, Completed, Archived. |

### Campaign

A Campaign is a focused, time-bound communication push around an event, season, message, series, launch, or call to action.

It answers:

> What are we promoting, explaining, launching, or moving people toward right now?

Examples:

- Promote Summer Camps.
- Promote VBS.
- Series Support: Advent.
- Easter Services.
- Christmas Eve Services.
- Fall Small Groups Launch.
- Volunteer Recruitment Month.
- Missions Giving Push.

Campaigns are usually:

- Time-bound.
- Message-driven.
- Audience-focused.
- Made up of one or more projects.
- Connected to a ministry need, event, season, sermon series, launch, or call to action.
- Tactical but still strategic.

Recommended Campaign fields:

| Field | Purpose |
|---|---|
| Name | Campaign name. |
| Goal | What the campaign should accomplish. |
| Audience | Who the campaign is trying to reach. |
| Desired action | What people should do. |
| Key message | Main communication idea. |
| Tone | Desired feeling. |
| Stakeholders | People involved. |
| Start date | When the communication effort begins. |
| End date | When the communication effort ends. |
| Major dates | Event, registration, launch, or response deadlines. |
| Related initiative | Optional strategic parent. |
| Ideas | Possible creative approaches. |
| Campaign brief | Strategic communication context. |
| Status | Planned, Active, Complete, Archived. |

### Program / Content Stream

A Program or Content Stream is a recurring operational rhythm that generates repeated projects or deliverables.

It answers:

> What recurring production rhythm does this work belong to?

Examples:

- Weekly Podcast.
- Weekly Sermon Clips.
- Weekly Email Newsletter.
- Sunday Service Slides.
- Monthly Missions Update.
- Campus Announcement Loop.

A Program / Content Stream is not the same thing as a Campaign.

For example, Weekly Sermon Clips is not a Campaign. It is a recurring content stream. A sermon clip may support a Campaign such as Series Support: Advent, but its primary production rhythm may still belong to the Weekly Sermon Clips content stream.

Recommended Program / Content Stream fields:

| Field | Purpose |
|---|---|
| Name | Program or stream name. |
| Purpose | Why the recurring work exists. |
| Cadence | Weekly, monthly, seasonal, etc. |
| Owner | Who oversees the stream. |
| Default audience | Common audience. |
| Default deliverables | Typical recurring outputs. |
| Default workflow notes | Typical production process. |
| Related initiative | Optional strategic parent. |
| Status | Active, Paused, Archived. |

### Sermon Series, Season, Ministry Area, Topic, and Tags

Some work may align to other strategic or organizational contexts, such as:

- Sermon Series.
- Advent.
- Lent.
- Summer.
- Children’s Ministry.
- Students.
- Missions.
- Volunteers.
- Hope.
- Stewardship.

These contexts may not need full first-class modules in MVP. They can begin as tags, categories, or relationship links depending on implementation needs.

## Operational Work

Operational work is where production is managed.

### Project

A Project is a defined body of work with an owner, objective, brief, start date, and stop date.

It answers:

> What body of work needs to be completed?

Projects are the primary operational container.

Every Project should require:

- Title.
- Owner.
- Department/ministry.
- Objective.
- Start date.
- Stop date.
- Status.

A Project may optionally relate to:

- Initiative.
- Campaign.
- Program / Content Stream.
- Sermon Series.
- Season.
- Ministry Area.
- Topic or tag.

Project date rule:

> Every Project must have a start date and stop date. The start date defines when work begins or is expected to begin. The stop date defines when the project work should be complete. The stop date is not always the same as the publish date, event date, or closeout date.

### Deliverable

A Deliverable is a specific output produced inside a Project.

It answers:

> What exactly needs to be made, reviewed, approved, scheduled, published, delivered, printed, installed, or archived?

Every Deliverable should belong to one Project as its primary operational home.

A Deliverable may also support or be associated with multiple strategic contexts through tags or relationship links.

Examples:

- Edited podcast episode.
- Show notes.
- Sermon bumper.
- Advent title slide.
- Social post.
- Email.
- Bulletin blurb.
- Website page.
- Print piece.
- Signage.

### Task

A Task is a specific execution step.

It answers:

> What specific thing needs to be done, by whom, and by when?

Tasks should belong to a Project and may also belong to a Deliverable.

Project-level tasks are for coordination.

Deliverable-level tasks are for production.

## Primary Home vs Strategic Associations

A Project or Deliverable should have one primary operational home, but may support multiple strategic contexts.

### Primary operational home

This answers:

> Where is the work managed?

For Deliverables, the primary home is always the Project.

### Strategic associations

This answers:

> What else does this work support or relate to?

A Project or Deliverable may be associated with:

- Initiative.
- Campaign.
- Program / Content Stream.
- Sermon Series.
- Season.
- Ministry Area.
- Topic.
- Tag.

This distinction prevents the model from becoming too rigid while still preserving clear ownership.

## Example: Series Support and Weekly Sermon Clips

Weekly Sermon Clips is a Program / Content Stream, not a Campaign.

Series Support: Advent is a Campaign.

A weekly sermon clip project may live operationally under the Weekly Sermon Clips content stream, while also supporting the Series Support: Advent campaign.

Example:

```text
Initiative:
  Increase digital sermon engagement by 50% this year

Program / Content Stream:
  Weekly Sermon Clips

Campaign:
  Series Support: Advent

Project:
  Advent Week 1 Sermon Clips

Deliverables:
  - Vertical sermon reel
  - YouTube Short
  - Facebook caption

Strategic associations:
  - Campaign: Series Support: Advent
  - Initiative: Increase digital sermon engagement by 50%
  - Sermon Series: Advent
  - Season: Advent
```

## Example: Weekly Podcast and Seasonal Alignment

A weekly podcast is a Program / Content Stream.

Each episode may be a Project.

Example:

```text
Program / Content Stream:
  Weekly Podcast

Project:
  Podcast Episode 31 - Hope in Exile

Deliverables:
  - Edited podcast audio
  - Show notes
  - Episode artwork
  - Website episode page
  - Social promo post
  - Email/newsletter blurb

Strategic associations:
  - Campaign: Series Support: Advent, if aligned
  - Sermon Series: Advent, if aligned
  - Season: Advent, if aligned
  - Topic: Hope
```

The podcast project remains operationally managed under the Weekly Podcast stream, but the episode or its deliverables may still appear in Advent-related reporting or campaign views through strategic associations.

## Example: Promote VBS

Promote VBS is a Campaign.

It may or may not support a larger Initiative.

Example:

```text
Campaign:
  Promote VBS

Projects:
  - Recruit Volunteers
  - Recruit Students / Families
  - VBS Production

Deliverables for Recruit Volunteers:
  - Volunteer recruitment email
  - Sunday announcement slide
  - Social post
  - Ministry leader talking points

Deliverables for Recruit Students / Families:
  - Parent email
  - Social post series
  - Website registration update
  - Promo video or reel
  - Bulletin/newsletter blurb

Deliverables for VBS Production:
  - Check-in signage
  - Room signs
  - Volunteer schedule handout
  - Service slides
  - Campus map
```

## Brief Cascade

Campaigns, Projects, and Deliverables may each carry brief information at different levels of specificity.

The cascade is:

```text
Strategic Context
  -> Project Brief
    -> Deliverable Brief
      -> Tasks
```

When Campaigns or Initiatives are present, they provide strategic context that flows into the Project Brief.

### Initiative Brief

Defines the broad measurable outcome.

Includes:

- Strategic goal.
- Success metric.
- Baseline.
- Target.
- Timeframe.
- Owner.
- Strategy notes.

### Campaign Brief

Defines the focused communication push.

Includes:

- Campaign goal.
- Audience.
- Desired action.
- Key message.
- Tone.
- Stakeholders.
- Campaign dates.
- Ideas.
- Outcomes.

### Program / Content Stream Brief

Defines recurring production purpose and rhythm.

Includes:

- Purpose.
- Cadence.
- Default audience.
- Default deliverables.
- Default workflow notes.
- Owner.

### Project Brief

Translates strategic context into an operational creative objective.

Includes:

- Project objective.
- Audience subset.
- Desired action.
- Scope.
- In-scope and out-of-scope work.
- Timeline.
- Stakeholders.
- Deliverable strategy.
- Approval plan.

### Deliverable Brief

Translates the Project Brief into production-specific direction for a concrete output.

Includes:

- Deliverable purpose.
- Output type.
- Audience.
- Desired action.
- Key message.
- Specs.
- Channel/platform.
- Owner.
- Due date.
- Publish/send/show/print/install date.
- Approval needs.
- Vendor needs.
- Source/inherited context.
- Deliverable-specific notes.

### Tasks

Tasks are execution steps derived from Project or Deliverable context.

Tasks should not carry strategy unless the detail is needed to complete the task.

## MVP Recommendation

For MVP:

- Projects are the required operational container for Deliverables.
- Deliverables should not stand alone.
- Projects must have a start date and stop date.
- Campaigns should remain optional strategic context, not the required parent for every Project.
- Initiatives should be optional strategic context.
- Program / Content Stream should be documented as an important concept, with simple tags or associations acceptable before full module support.
- Strategic associations may begin as tags or relationship links before becoming full modules.
- The user experience should not require department leaders to understand these distinctions during intake.

## Future Expansion

Future versions may add first-class support for:

- Initiatives.
- Campaign planning dashboards.
- Program / Content Stream templates.
- Recurring project generation.
- Sermon series planning.
- Seasonal content grouping.
- Cross-context reporting.
- Deliverable association across campaigns, initiatives, programs, topics, and seasons.

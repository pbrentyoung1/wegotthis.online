# Project Closeout

## Purpose

Document the structured closeout process that prepares completed communications work for archive, reuse, reporting, and institutional memory.

## Current Decisions

- Working direction: closing a project should be a workflow, not simply marking it done.
- Working direction: projects should move from Published to Closeout before Archived.
- Working direction: closeout should connect final assets to the Media Library where appropriate.

## Closeout Workflow

- Verify final approved deliverables.
- Confirm source files exist.
- Confirm exported files exist.
- Save vendor-ready files.
- Archive project notes.
- Archive approvals/sign-offs.
- Archive budget/cost details.
- Archive final timeline.
- Connect all final assets to the media library.
- Tag reusable assets.
- Record where assets were used.
- Mark files as final, draft, source, export, or vendor-ready.
- Capture lessons learned.
- Move project from Closeout to Archived.

## Recommended Archive Folder Structure

```text
/archive
  /year
    /department
      /project-name
        /01-final-deliverables
        /02-source-files
        /03-exports
        /04-print-and-vendor
        /05-web-and-social
        /06-photos-and-video
        /07-copy-and-messaging
        /08-approvals-and-notes
```

## File Statuses

- Final: approved deliverable intended for use or publication.
- Draft: working file or review version not approved as final.
- Source: editable native file used to create deliverables.
- Export: generated output for web, social, print, review, or archive.
- Vendor-ready: file prepared for print, signage, production, installation, or external vendor use.

# Media Asset Workflow

## Purpose

Define how media assets are added, tagged, approved, updated, and reused.

## Current Decisions

- Media asset management is a core product area.
- Media assets should include metadata such as tags, version, notes, status, uploader, time added, usage, people in image, and context.

## Open Questions

- Who can upload assets?
- Do assets require approval before use?
- How are asset versions replaced or archived?

## Notes

- Asset workflow should support both project files and reusable brand/media resources.

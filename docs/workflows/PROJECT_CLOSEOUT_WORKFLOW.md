# Project Closeout Workflow

## Purpose

Document a practical closeout workflow for moving completed projects into a searchable, reusable archive.

## Current Decisions

- Working direction: projects should enter Closeout before Archived.
- Working direction: closeout should verify deliverables, assets, metadata, approvals, budget details, and reusable context.

## Step 1: Confirm Completion

- [ ] Final deliverables are approved.
- [ ] Publication, delivery, print, installation, or production is complete.
- [ ] Stakeholders know the project is entering closeout.

## Step 2: Verify Files

- [ ] Final deliverables exist.
- [ ] Source files exist where applicable.
- [ ] Exported files exist.
- [ ] Vendor-ready files exist where applicable.
- [ ] Web and social exports exist where applicable.
- [ ] Photos and video files are accounted for.

## Step 3: Verify Metadata

- [ ] Files are marked as final, draft, source, export, or vendor-ready.
- [ ] Reusable assets are tagged.
- [ ] Asset usage is recorded.
- [ ] People, location, ministry, event, and context are captured where useful.

## Step 4: Archive Project Context

- [ ] Project notes are saved.
- [ ] Approvals and sign-offs are saved.
- [ ] Budget and cost details are saved.
- [ ] Final timeline is saved.
- [ ] Lessons learned are captured.

## Step 5: Connect to Media Library

- [ ] Final assets are linked to the project.
- [ ] Reusable assets are added or promoted to the Media Library.
- [ ] Brand assets are identified where applicable.
- [ ] Usage history is recorded.

## Step 6: Archive Package

- [ ] Files follow the approved archive structure.
- [ ] Missing items are documented.
- [ ] Exceptions are approved by the right role.
- [ ] Project is ready to move to Archived.

## Step 7: Archive

- [ ] Project status moves from Closeout to Archived.
- [ ] Archived project remains searchable.
- [ ] Reports can include project volume, time, budget, and department usage.
